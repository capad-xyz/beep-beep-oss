### The mental model
1. React calls a function in `src/api.ts`.
2. That calls Tauri's `invoke("command_name", args)`.
3. Tauri routes it to the matching `#[tauri::command]` in `src-tauri/src/matrix.rs`.
4. Rust does the Matrix work and returns a serializable struct.
5. The struct's TypeScript type is auto-generated by **ts-rs**, so the two sides
   can never silently disagree about the shape of the data.

That generation step is the fix for the "two languages drift apart" gotcha we
worried about when picking this stack.

## Prerequisites

- **Rust** (`rustup`)
- **Node.js + npm**
- **Tauri 2 system deps** вЂ” follow https://tauri.app/start/prerequisites/ for your OS
  (webview libs, build tools).
- **App icons** вЂ” `src-tauri/icons/` must contain icons before a release build.
  Generate them with `npm run tauri icon path/to/logo.png`, or scaffold a throwaway
  app with `npm create tauri-app@latest` and copy its `icons/` folder.

## Running it (dev)

```bash
cd client
npm install
npm run tauri dev
```

This starts Vite (frontend) and the Rust app, opening a window. Sign in against a
homeserver вЂ” e.g. your Phase 0 stack at `http://localhost:8008` (see ../SETUP.md).

## Regenerating the TS bindings

The types in `src/bindings/` come from the Rust structs marked `#[derive(TS)]`.
After changing such a struct, regenerate:

```bash
cd src-tauri
cargo test        # ts-rs writes the .ts files during its export test
```
